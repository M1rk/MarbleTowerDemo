# Marble Tower Demo - Project Context

## Цель прототипа
Сделать кооперативный прототип, где игроки с разных компьютеров могут:
- подключиться в лобби,
- запустить игру,
- собраться на базе,
- уйти в экспедицию,
- выполнить простые активности,
- вернуться на базу.

Важно: в MVP нет системы постоянных сохранений (persistent save).

## Уровни
- `FortressLevel`: база/крепость. Здесь управление ресурсами, выбор экспедиции, развитие и подготовка.
- `ExpeditionLevel`: зона вылазки, где добываются ресурсы и энергия для эвакуации.

## Сеттинг
- Средневековый антураж.
- Игроки сосланы в удалённую крепость на границе диких земель.
- В мире мало твёрдого камня, ценен мрамор из руин древней цивилизации.

## Базовый игровой цикл
1. Сбор группы в `FortressLevel`.
2. Выбор миссии и переход в `ExpeditionLevel`.
3. Добыча ресурсов (в т.ч. мрамора) и энергии.
4. Запуск ритуала возврата через портал.
5. Удержание позиции до завершения прогресс-бара эвакуации.
6. Возврат в `FortressLevel`.

## Эвакуация и риск
- Эвакуация идёт через портал.
- Чем больше вес отправляемых вещей, тем больше нужно энергии.
- Для эвакуации самих персонажей тоже нужна энергия.
- Со временем миссии среда становится агрессивнее.
- Во время ритуала идут волны врагов, они могут сорвать выход.

## Ожидания от MVP
- Простой, но рабочий кооп host/client (хост хранит состояние сессии).
- Минимальный набор активностей в экспедиции: например рыбалка, охота, добыча в реке.
- Базовые враги (упрощённые по сравнению с финальным видением).
- Рабочий цикл "база -> экспедиция -> эвакуация -> база".
